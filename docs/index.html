Introduction

FlaskNova is a productivity-first framework extension built for the modern Python era. It was designed with a single goal: to make API development in Flask as seamless and automated as possible.

Core Tech: Pydantic V2 FastAPI Inspired OpenAPI 3.1

Heavily inspired by the developer experience of FastAPI, FlaskNova brings modern features to the Flask ecosystem. By leveraging Pydantic for data validation, FlaskNova ensures your request payloads and response models are strictly typed, validated, and self-documenting.

Zero Migration Debt

No need to rewrite your entire legacy Flask app. Get FastAPI-style features while keeping your existing architecture intact.

ðŸ”Œ Ecosystem Friendly

Fully compatible with your favorite extensions like Flask-Login, Flask-Admin, and Flask-SQLAlchemy.

Blazing Performance

Powered by Pydantic V2, offering lightning-fast data validation and serialization optimized in Rust.

Development shouldn't involve manually updating JSON schemas or writing tedious validation logic for every route. FlaskNova handles the heavy lifting, providing automatic OpenAPI documentation, type-safe routing, and built-in serialization. It transforms Flask from a micro-framework into a robust, full-featured API engine without losing the simplicity that Flask developers love.

Installation

Install the core package via pip:

pip install flask-nova

Getting Started

1. Basics: Simple GET Request

Define your routes using the @app.get decorator. FlaskNova automatically populates your documentation UI.

Docstring Support

If you don't provide a summary or description, FlaskNova intelligently parses your docstring: The first line becomes the Summary, and the following lines become the Description.

@app.get("/welcome")
def welcome():
    """Welcome User.

    This text will show up as the long description in Swagger/Redoc.
    """
    return {"message": "Welcome to FlaskNova!"}

2. Respecting Flask Internals

FlaskNova doesn't take away Flask's internals. You can still use make_response, jsonify, and custom headers exactly as you do in standard Flask. 

from flask import make_response, jsonify

@api.route("/custom", methods=["GET"])
def custom():
    data = {"message": "Custom response"}
    response = make_response(jsonify(data), 201)
    response.headers['X-Custom-Header'] = 'Value'
    return response

3. Request/Response Handling

Use Pydantic models for validation and serialization. For form data, use the Form() parser.

from flasknova import Form, status
from pydantic import BaseModel

class UserSchema(BaseModel):
    username: str
    age: int

@app.post("/register", response_model=UserSchema)
def register(data: UserSchema = Form(UserSchema)):
    # Returns 201 Created using semantic status
    return data, status.CREATED
status.OK (200)
status.CREATED (201)
status.BAD_REQUEST (400)
status.UNAUTHORIZED (401)

Custom Classes

Note: When using custom classes for responses, you must implement a to_dict() method to map how you want the data to be serialized. 
class MyResponse:
    def __init__(self, val):
        self.val = val

    def to_dict(self):
        return {"value": self.val}

4. Why Pydantic?

Unlike standard Flask dictionaries, Pydantic offers:

  • Automatic Validation: Invalid data is rejected with 422 errors before reaching your code.
  • IDE Support: Full autocomplete and type hinting.
  • Data Casting: Automatically converts "123" to 123 for integer fields.

Advanced Features

Dependency Injection (Depend)

Modularize your logic by injecting dependencies directly into your function signatures.

from flasknova import Depend

def get_token_header(x_token: str = "secret"):
    return x_token

@app.get("/secure")
def secure_route(token = Depend(get_token_header)):
    return {"token": token}

The @guard Decorator

Combine multiple security layers or filters into one clean decorator. Avoid: Decorator stacking which makes code hard to read. 

from flask_jwt_extended import jwt_required
from flask_nova import guard

@app.get("/admin")
@guard(jwt_required(), admin_only_check)
def admin_dashboard():
    return {"data": "confidential"}

Error Handling & Color Logging

FlaskNova uses a robust HTTPException class to handle errors consistently.

from flasknova import HTTPException, status

raise HTTPException(
    status_code=status.NOT_FOUND,
    detail="User not found",
    title="Resource Error"
)

CLI Mastery

Flask-Nova provides a powerful CLI tool to automatically generate artifacts for your routes.

Command

flask-nova gen --app <your_app_path> [OPTIONS]

Options

Option Description
--app TEXT Required. App path, e.g. examples.form_ex:app
--format [http|py|all] File format to generate (default: all)
--base-url TEXT Base URL for requests (default: http://127.0.0.1:5000)
--output PATH Directory to save files (default: current directory)
Generate both HTTP and Python requests:
$ flask-nova gen --app main:app --format all
[SCAN] 14 routes detected...
[DONE] Created 14 .http requests in ./docs/requests
[DONE] Created 14 pytest scripts in ./tests/gen
Generate HTTP requests only flag --output path:
$ flask-nova gen --app examples.form_ex:app --format http --rest
Generate Python requests only:
$ flask-nova gen --app examples.form_ex:app --format py

Project Evolution

FlaskNova has moved fast, evolving from a core routing engine to a full-featured API framework in record time.

  • v0.1.0: The Foundation CORE

    The birth of the Core Routing Engine. Launched with automatic OpenAPI schema generation, deep Pydantic integration for typed URL parameters, and customizable metadata support.

  • v0.1.1: Security & Logic ENHANCEMENT

    Introduction of the @guard() decorator for clean middleware handling and Form() parsing for complex payloads. Streamlined configuration by removing set_swagger() in favor of automated discovery.

  • v0.1.2: Full API Spectrum ECOSYSTEM

    Expanded route handling to support the full REST spectrum: GET, POST, PATCH, PUT, and DELETE. Added ReDocs support alongside Swagger for beautiful, alternative documentation.
